## 五、关键洞察

1. **等 shares 策略**：该账号买近似等量的 Up 和 Down shares。如果 P_up + P_dn < 1.0，无论结果如何都赚。这是经典的**条件代币套利**。

2. **并非纯套利**：平均 edge 在 BTC 5min 上仅 +0.59%，且有 43% 的窗口 edge 为负。说明他不是只在有 edge 时才入场。

3. **大资金 + 拆单**：每个窗口投入数千到上万美金，拆成上百笔小单。这可能是为了：
   - 避免滑点（不一次吃光 orderbook）
   - 分批获取更好的均价
   - 适应 orderbook 的实时变化

4. **存在方向偏斜**：share ratio 在某些窗口偏离 1.0 很多（0.63~1.67），暗示可能有信号驱动的方向偏好，但不像我们那样固定 80/20。

5. **可能的盈利模型**：
   - 在 P_up + P_dn < 1.0 时大量入场（正 edge 套利）
   - 在 P_up + P_dn ≈ 1.0 时也入场（赌方向 + 结算结果的期望值）
   - 通过大资金量 × 小 edge 赚取绝对收益

## 六、对我们策略的启示

1. **分配方式**：考虑改为**等 shares 分配**而非等 USDC 分配，这样两侧 payout 相等

4. **资金规模**：该账号每窗口投入是我们的 400~1000 倍，说明此策略需要规模效应
